Development Phases

Three Phases of SignBridge

Each phase builds on the last — from prototype to a full-featured communication platform.

Phase 1 Foundation

📷 Core Prototype

Camera feed, MediaPipe hand skeleton overlay, ASL fingerspelling detection, and a chat shell with basic text/speech modes.

  • Live webcam with hand skeleton overlay
  • MediaPipe Hands landmark detection
  • ASL A–Z fingerspelling classifier
  • Letter buffer → send as message
  • Speech-to-text (Web Speech API)
  • Text-to-speech for signed messages
  • Chat UI with sent/received bubbles

Phase 2 Expanded

🔮 Full Feature Set

200+ word ASL dictionary, SVG hand avatar animations, word prediction, hold-time guidance, and a persistent translated message log.

  • 200+ word ASL sign dictionary
  • Word prediction strip (up to 6 chips)
  • Hold-time guidance bar with color steps
  • SVG hand avatar animated word-by-word
  • Translated message log on left panel
  • Sign reference card (letter + handshape)
  • Quick-sign chips for common phrases
  • Full settings drawer (hold, TTS, speed)

Phase 3 Advanced

🚀 Production Ready

50/50 split layout, NLP grammar correction, emotion detection, sign training mode with scoring, and room-sharing UI.

  • True 50/50 split panel layout
  • NLP grammar correction (ASL→English)
  • Emotion detection from facial expression
  • Sign Training Mode with live scoring
  • Room sharing (WebRTC-ready UI)
  • Enhanced 3D-style hand avatar renderer
  • Tab bar (Chat / Training / Stats)

Technology

Quick Start

Get Running in 30 Seconds

No npm, no build tools, no dependencies to install.

# Clone the repository
git clone https://github.com/your-username/signbridge.git
cd signbridge
# Open directly in your browser (no server needed)
open phases/phase3.html
# Or serve locally for camera access
npx serve .
⚠️ Camera Note: Browsers require HTTPS or localhost for camera access. Use npx serve . or deploy to any static host (GitHub Pages, Netlify, Vercel).
